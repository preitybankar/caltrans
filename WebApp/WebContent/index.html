RUSLE Application



The Stormwater industry in California needs an application that will easily determine the soil loss (erosion) inputting existing conditions for construction project. The application will be developed to comply with the requirements of the California General NPDES Permit (or Construction General Permit (CGP)) for stormwater discharges associated with construction activity.


The basis for the application is the Revised Universal Soil Loss Equation (RUSLE):

A = R x K x LS x C x P


Where:

  • A = Soil Loss in tones/acre/year
  • R = Erosivity (based on schedule and rainfall intensity)
  • K = Erodibility (based silt content of soil)
  • LS = Length/Steepness (based on representative slope length and steepness)
  • C = Cover Management (BMPs based on cover effectiveness)
  • P = Erosion Control Practice (other erosion control BMP effectiveness)


The post-construction soil loss needs to be equal to or less than pre-construction. The application will compare the pre-construction soil loss to the post-construction soil loss.

The RUSLE 2 program allows multiple “transects” to be entered. This means that there may be several inputs for representative areas on a construction project.


The application accepts input for the multiple areas and compute a weighted average based on area or percentage represented by the transect.The application will also be able to help model BMPs during construction. Input of during construction BMPs would also be compared to pre-construction conditions to determine if the BMPs are adequate (less than or equal to pre-construction soil loss).


The application will need to have a data base for the input parameters. The R factor is based on a schedule and rainfall intensity. There is a rain fall intensity map with contours that would need to be included. There is a table for areas to determine percentages of rainfall during the year. Then the percentages of the year (based on schedule input) would be multiplied by the annual intensity (based on location on the map contours)


There is a map of K values. Based on location the correct K value could be defaulted. But this should include an input box for measured K. These are typically 0.24 to 0.40.


The LS factor would be based on a table of length versus slope percent. The application would need to take the input and output a factor based on the table.


The C factor would need a list of cover BMPs with associated effectiveness to choose from and an input for percent or area of coverage.


There should also be an “other” to enter project specific BMPs and their effectiveness. I would assume the list included in the app could grow over time based on feedback for additional BMPs.


The P factor similarly would need a list of BMPs with effectiveness to select one or more with a percentage or area.


This is not just a replacement of the RUSLE 2 program. This is to be more specific to the needs of CA construction. The RUSLE was developed from a farming aspect to reduce soil loss from wind dating back to the Dust Bowl. So, there may be many farming aspects that do not apply. For example, a typical construction BMPs does not include fallen row crops such as corn but should include straw mulch that could be applied to the disturbed soil area to cover it at a 2-inch thickness.


We can work on getting the best BMP effectiveness data from the Caltrans website, RUSLE 2 program, SMARTS database (Water Boards), etc. In fact, we might want to share this with CASQA once we get it started for input.
